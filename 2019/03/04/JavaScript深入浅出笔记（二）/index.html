with语句

不建议使用with语句：

1、让js引擎优化更难

2、可读性差

3、可被变量定义代替

4、严格模式下被禁用

严格模式

严格模式是一种特殊的执行模式，它修复了部分语言上的不足，提供更强的错误检查，并增强安全性

严格模式有如下特点

1、 不允许使用with

2、不允许未声明的变量被赋值

3、arguments变为参数的静态副本

在一般模式下，如果定义了一个函数，调用它并且传递参数，那么它对应的形参和arguments[0]有一个相互的绑定关系，就是说如果我们修改了arguments[0]，那么这个函数对应的形参a就会被修改了，所以输出100。

如果不给函数传值，那么无论我们如何修改arguments[0]的值，输出的都是undefined，因为没有传递参数进去

但是在严格模式下：arguments[0]变为参数的静态副本，就是说无论函数的参数有没有传递，都不会和arguments相互影响

但是如果传入的是一个对象的话，将会按照共享传递，使用arguments修改对象的属性，那么还是会相互影响的。
